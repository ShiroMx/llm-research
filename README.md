## Словник термінів (Глосарій)

* **Великі мовні моделі** (Large Language Models, LLM) — тип штучного інтелекту, навчений на величезних масивах тексту для розуміння та генерації людської мови.
* **Контекстне вікно** (Context Window) — обсяг інформації (тексту, коду, документів), який модель може "тримати в пам'яті" та аналізувати одночасно під час одного запиту. Вимірюється в токенах.
* **Токен** (Token) — базова одиниця тексту для LLM. Це може бути частина слова, ціле слово або символ. Орієнтовно: 1000 токенів ≈ 750 слів англійською.
* **Конституційний ШІ** (Constitutional AI, CAI) — метод навчання моделей, розроблений Anthropic, де модель тренується оцінювати та коригувати свої відповіді згідно з набором прописаних принципів ("конституції"), замість покладання лише на ручну розмітку людьми.
* **RLHF / RLAIF** (Reinforcement Learning from Human/AI Feedback) — методи донавчання моделей через підкріплення, де "вчителем" виступає людина (Human) або інша модель ШІ (AI). Anthropic активно використовує RLAIF.
* **Агентний ШІ** (Agentic AI) — здатність моделі не просто відповідати на питання, а виконувати послідовність дій (використання комп'ютера, написання та запуск коду, керування файлами) для досягнення складної мети.
* **Галюцинації** (Hallucinations) — явище, коли модель генерує граматично правильну, але фактично неправдиву або нісенітну інформацію.
* **Artifacts** (Артефакти) — елемент інтерфейсу користувача (UI), який дозволяє відокремлювати згенерований контент (код, діаграми, веб-сторінки) від текстового чату для зручного перегляду та редагування. Не плутати з самою моделлю.

## 1. Вступ: Anthropic та філософія "Безпечного ШІ" (AI Safety)

Компанія **Anthropic** займає позицію головної етичної противаги в гонці озброєнь ШІ. Заснована у 2021 році вихідцями з OpenAI, компанія будує свою ідентичність навколо концепції **"Helpful, Harmless, and Honest" (Корисний, Нешкідливий, Чесний)**.

Anthropic позиціонує себе як *Public Benefit Corporation (PBC)*. Їхня стратегія відрізняється фокусом на:
1.  **Interpretability (Інтерпретованість):** Розуміння того, *як* і *чому* нейромережа приймає рішення.
2.  **Long Context:** Ставка на величезні контекстні вікна для роботи з книгами та базами коду.
3.  **Safety First:** Використання Constitutional AI для створення більш передбачуваних моделей.

## 2. Еволюція моделей Claude

Історія розвитку Claude демонструє перехід від "безпечного чат-бота" до найпотужнішого інструменту для розробників.

### 2.1. Ранні версії
* **Claude 1 (Березень 2023):** Перший конкурент GPT-3.5. Фокус на безпеці.
* **Claude 2 / 2.1 (Липень-Листопад 2023):** Впровадження контексту **100K** та **200K** токенів. Це зробило Claude стандартом для аналізу великих документів (PDF, книги).

### 2.2. Сімейство Claude 3
Березень 2024 року. Розділення на три класи:
* **Haiku:** Швидкість та дешевизна.
* **Sonnet:** Баланс (Enterprise стандарт).
* **Opus:** Максимальний інтелект (перший "вбивця" GPT-4).

### 2.3. Прорив Claude 3.5
* **Claude 3.5 Sonnet (Червень 2024 / Жовтень 2024):** Модель середнього розміру, яка перевершила флагманів конкурентів. Ввела **Computer Use** (керування ПК). Стала "золотим стандартом" для кодингу.
* **Claude 3.5 Haiku (Жовтень 2024):** Найшвидша модель, яка за інтелектом зрівнялася з попереднім флагманом (Claude 3 Opus).

### 2.4. Claude 3.7 та Hybrid Reasoning
* **Claude 3.7 Sonnet (Лютий 2025):** Перша модель з гібридним мисленням. Користувач може обирати між миттєвою відповіддю або режимом **"Extended Thinking"** (розширене мислення), де модель витрачає час на побудову ланцюжка роздумів перед відповіддю. Це значно покращило результати в математиці та складній логіці.

### 2.5. Сімейство Claude 4 та 4.5
* **Claude 4 (Травень 2025):** Нова архітектура з ще глибшим розумінням контексту.
* **Claude 4.5 Sonnet / Opus (Вересень-Листопад 2025):** Актуальні SOTA (State-of-the-Art) моделі станом на кінець 2025 року. Характеризуються майже повною відсутністю галюцинацій у робочих задачах та здатністю вести автономну розробку програмного забезпечення (Agentic Coding) протягом годин без втручання людини.

## 5. Бенчмарки та порівняння ефективності

Станом на кінець 2025 року, Anthropic зберігає лідерство в категорії **Coding & Reasoning**.

* **SWE-bench Verified (Engineering):** Claude 4.5 Sonnet демонструє здатність вирішувати **>65%** реальних задач з GitHub автономно (порівняно з ~49% у версії 3.5).
* **MATH / GPQA (Science):** Завдяки режиму "Extended Thinking", моделі 3.7 та 4.5 досягають точності понад 90% у завданнях олімпіадного рівня, конкуруючи лише з спеціалізованими моделями OpenAI (o-series).

## 6. Інструменти та підходи Anthropic для розробки

Важливо розрізняти самі **моделі** (Claude Sonnet/Opus) та **інструменти/сервіси**, які допомагають з ними взаємодіяти.

### 6.1. Constitutional AI (Конституційний ШІ)
Технологія "самонавчання" безпеки. Модель отримує "Конституцію" (набір правил) і використовує її для критики власних відповідей під час тренування (RLAIF). Це зменшує потребу в людському втручанні.

### 6.2. Interpretability (Інтерпретованість)
Науковий підхід до "відкриття чорної скриньки". Інженери Anthropic навчилися ідентифікувати групи нейронів, що відповідають за конкретні концепти (міста, емоції, загрози), і можуть "вмикати" або "вимикати" їх вручну.

### 6.3. Artifacts (Інтерфейс користувача)
**Це не модель, а UI-функція.**
Введена у 2024 році, функція **Artifacts** дозволяє відображати згенерований контент (код React, SVG-графіку, документи Markdown) у окремому інтерактивному вікні поруч з чатом. Це перетворює спілкування з Claude з режиму "питання-відповідь" у режим "спільна робота над документом/додатком".

### 6.4. Prompt Caching
API-інструмент для розробників, що дозволяє "кешувати" (зберігати в пам'яті сервера) великі обсяги вхідних даних (наприклад, всю документацію проекту). Це знижує вартість запитів на 90% і пришвидшує відповідь моделі.

### 6.5. Claude Code (CLI Agent)
**Окремий продукт для розробників (Bonus Research).**
Це автономний інструмент командного рядка (CLI), випущений Anthropic. Він дозволяє Claude працювати безпосередньо в терміналі розробника.

**Можливості Claude Code:**
* **Доступ до файлової системи:** Може читати, редагувати та створювати файли у вашому проекті.
* **Робота з Git:** Вміє самостійно робити коміти, вирішувати конфлікти злиття (merge conflicts) та створювати Pull Requests.
* **Запуск тестів:** Може запустити команду (наприклад, `npm test`), проаналізувати помилки та самостійно виправити код, повторюючи цикл до успіху.
* **Agentic Workflow:** На відміну від звичайного чату, Claude Code може скласти план дій з 10 кроків і виконувати їх послідовно, звітуючи про прогрес.
